## 输出要求
输出一个 JSON 对象，包含完整的大纲层级。严格按照以下结构：

```json
{
  "arcs": [
    {
      "title": "总纲标题（一句话概括全书核心冲突与主题）",
      "summary": "总纲概述（2-3句话描述故事主线与结局方向）",
      "volumes": [
        {
          "title": "卷标题（一句话概括该卷核心冲突弧）",
          "summary": "卷概述（2-3句话描述该卷主线与关键转折）",
          "chapters": [
            {
              "title": "章标题（本章核心事件）",
              "summary": "本章概述（本章发生了什么，对主线有何推动）",
              "scenes": [
                {
                  "title": "场景标题",
                  "summary": "场景概述（谁在哪儿做了什么，本场景在大纲中的功能）"
                }
              ]
            }
          ]
        }
      ]
    }
  ]
}
```

规则（广度优先，先把骨架铺完整再往下展开）：

- **arcs：至少 3 个**——必须覆盖完整故事弧（开局 / 发展 / 高潮 / 收尾，可拆为 3-5 个 arc）。
  不是只写第一卷的 1 个 arc + 三层下钻，而是把全书主线先铺平。
- **每个 arc 下至少 2 个 volume**，体现该 arc 内部的明显转折。
- **每个 volume 下至少 2 个 chapter**——一个孤本 chapter 通常意味着 arc/volume 切分太细。
- 每章 2-4 个场景（仅 scripted 模式；anchor-only 模式按"模式"段说明强制空数组）。
- 各 arc 之间字数要均衡：不要第一个 arc 写得很详（多 volume 多 chapter），后面 arc 只剩光秃秃的 1 个 volume。**先广度，后深度**。
- 所有 title 和 summary 必须用中文。
- summary 不可为空字符串，描述要有信息量（点出本节的核心冲突 / 转折 / 关键事件，至少 30 字）。
- 只输出 JSON，不要任何其他文字或解释。
