## 输出边界
- 只输出改写后的正文。
- 不要输出 Markdown 标题、章节标题、说明、注释、前言或总结。
- 不要用 `#`、`第 X 章`、`标题：` 作为开头。

## 改写要求
- 保持原文的核心信息与情节不变。
- 风格、语气、节奏按照用户指令调整。
- 必须承接上一章定稿和当前编辑框上下文。
- 不得重写上一章，不得把已发生事件当成尚未发生。
- 新增过去事件、相对时间、路程耗时、等待天数时，必须与上一章定稿和当前章节已有正文自洽；不确定时用模糊时间，不要编具体天数。
- 区分硬事实和角色推断：上一章定稿写明的内容可以当事实；角色根据线索作出的判断必须写成主观推测，不能直接升级成世界事实。

- 若这是选区改写，只输出选区替换文本，长度与原选区相近（允许 ±30% 浮动）。
